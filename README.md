# TestClient
Android webSocket client

Главный класс - client.WebSocketConnection. Его методы можно переписать под свои нужды, и подключать для передачи серверу данных о заказах (в виде объектов Order),
получения списка зарегистрированных на сервере парковок, обновления парковок при изменении их состояния (числа свободных мест).
Отображение заказов на стороне парковок в процессе реализации, как и уведомление о регистрации заказов.

Класс Main создан для тестовых целей. Клиент получает сведения о всех зарегистрированных парковках, отправляет каждые 5 секунд случайный заказ на 
случайным образом выбранную парковку. Параметры отправки сообщений (требуемая парковка, время начала брони, частота отправка заказов и т.п.) могут 
регулироваться.

Классы в пакете objects желательно не менять, формат представления служебной информации (адреса парковки, стоимости, платежная информация в заказе и т.п) определяется клиентской стороной, вся служебная информация выносится в поле info.
